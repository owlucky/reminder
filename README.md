# Reminder Service — лёгкий сервис-напоминалка

Сервис отправляет напоминания о событиях (дни рождения, дедлайны задач и т.п.)
— разово или периодически, одному человеку или группе, за нужное время до
события (за неделю, за день, в момент). Доставка идёт через **подключаемые
каналы**; сейчас реализован **Telegram**, остальные (email, SMS) добавляются
несколькими строками кода.

## Возможности

- **Разовые и периодические** напоминания: `once` / `daily` / `weekly` /
  `monthly` / `yearly` (годовые удобны для дней рождения) с интервалом
  (каждые N периодов).
- **Несколько напоминаний до события**: список смещений, например
  «за неделю», «за день» и «в момент» → `["1w", "1d", "0"]`.
- **Получатели — люди и группы**: у напоминания может быть несколько
  получателей; групповой чат Telegram тоже просто получатель с `chat_id` группы.
- **Подключаемые каналы доставки**: единый интерфейс, добавить email/SMS/push —
  это новый класс + одна строка регистрации.
- **Полное отключение** (`disable`/`enable`) и **временная пауза**
  (`snooze`, по умолчанию на месяц).
- **Надёжность**: журнал отправок и дедупликация (одно и то же не отправится
  дважды), досыл пропущенного после простоя в пределах grace-окна.
- **Легковесность**: SQLite + FastAPI + APScheduler, запуск в Docker.

## Архитектура

```
app/
  main.py          — FastAPI, старт планировщика и каналов
  config.py        — настройки из окружения/.env
  database.py      — SQLAlchemy (по умолчанию SQLite)
  models.py        — Reminder, Recipient, SentNotification
  schemas.py       — Pydantic-схемы API
  recurrence.py    — парсинг смещений + расчёт моментов события
  dispatcher.py    — поиск наступивших напоминаний и рассылка
  scheduler.py     — фоновый опрос (APScheduler)
  telegram_bot.py  — интерактивный бот (long polling) для управления из чата
  channels/        — каналы доставки (base/registry/telegram/console/email/sms)
  routers/         — HTTP-эндпоинты
```

## Быстрый старт (Docker)

```bash
cp .env.example .env
# впишите TELEGRAM_BOT_TOKEN (получить у @BotFather)

docker compose up --build
```

Сервис поднимется на http://localhost:8000, интерактивная документация —
http://localhost:8000/docs. Данные SQLite сохраняются в томе `./data`.

## Быстрый старт (локально)

```bash
python -m venv .venv
. .venv/Scripts/activate        # Windows PowerShell: .venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Без токена Telegram можно всё протестировать через канал `console`
(уведомления печатаются в лог) — создайте получателя с `"channel": "console"`.

## Как это работает

1. Заводите **получателей** (`/recipients`) — кому и через какой канал слать.
2. Создаёте **напоминания** (`/reminders`) — что, когда, с какой периодичностью,
   за сколько до события и каким получателям.
3. Планировщик каждые `POLL_INTERVAL_SECONDS` секунд проверяет, для каких
   напоминаний наступил момент срабатывания (`момент_события - смещение`),
   и рассылает их, фиксируя в журнале, чтобы не повторяться.

## Примеры API

Получить `chat_id`: напишите своему боту, затем откройте
`https://api.telegram.org/bot<TOKEN>/getUpdates` — там будет `chat.id`.

### Получатель (Telegram)

```bash
curl -X POST http://localhost:8000/recipients \
  -H "Content-Type: application/json" \
  -d '{"name": "Иван", "channel": "telegram", "address": "123456789"}'
```

### Напоминание о дне рождения (ежегодно, за неделю и в день)

```bash
curl -X POST http://localhost:8000/reminders \
  -H "Content-Type: application/json" \
  -d '{
        "title": "День рождения Ивана",
        "event_time": "1990-05-05T09:00:00Z",
        "recurrence": "yearly",
        "offsets": ["1w", "0"],
        "recipient_ids": [1]
      }'
```

### Напоминание о дедлайне (разово, за день и за час)

```bash
curl -X POST http://localhost:8000/reminders \
  -H "Content-Type: application/json" \
  -d '{
        "title": "Дедлайн: отчёт",
        "description": "Сдать квартальный отчёт",
        "event_time": "2026-07-15T18:00:00Z",
        "recurrence": "once",
        "offsets": ["1d", "1h"],
        "recipient_ids": [1]
      }'
```

### Управление

```bash
# временно отключить (по умолчанию на месяц)
curl -X POST http://localhost:8000/reminders/1/snooze
curl -X POST http://localhost:8000/reminders/1/snooze -H "Content-Type: application/json" -d '{"days": 14}'

# снять паузу / полностью выключить / включить
curl -X POST http://localhost:8000/reminders/1/unsnooze
curl -X POST http://localhost:8000/reminders/1/disable
curl -X POST http://localhost:8000/reminders/1/enable

# тестовая отправка сейчас, ближайшие срабатывания, список каналов
curl -X POST http://localhost:8000/reminders/1/test
curl http://localhost:8000/reminders/1/upcoming
curl http://localhost:8000/channels
```

## Telegram-бот (управление из чата)

Если задан `TELEGRAM_BOT_TOKEN`, при старте поднимается интерактивный бот
(long polling, в фоновом потоке — без лишних зависимостей). Управлять
напоминаниями можно прямо из чата, `chat_id` определяется автоматически.
**Все выборы делаются inline-кнопками.**

При первом обращении (`/start` или `/add`) бот просит выбрать **часовой
пояс** — кнопками. После этого дату/время вы вводите **в своём поясе**, и
все напоминания и даты в сообщениях показываются тоже в нём (проблема
рассинхрона с UTC решена). Сменить пояс позже — `/timezone`.

Команды:

| Команда | Действие |
| --- | --- |
| `/start` | приветствие + выбор часового пояса при первом запуске |
| `/add` | пошагово создать напоминание (заголовок и дата — текстом; повтор и смещения — кнопками) |
| `/list` | напоминания этого чата, с кнопками управления под каждым |
| `/timezone` | выбрать/изменить часовой пояс (кнопками) |
| `/cancel` | отменить текущий пошаговый ввод |
| `/help` | справка |

Команды бота зарегистрированы через `setMyCommands`, поэтому доступны по
кнопке **Menu** слева от поля ввода — листать историю вверх не нужно.

Управление конкретным напоминанием — кнопками под ним в `/list`:
**📨 Тест**, **✏️ Изменить**, **⏸ Пауза**, **🚫 Выключить / ✅ Включить**,
**🗑 Удалить** (с подтверждением). Кнопка **✏️ Изменить** позволяет поменять
«когда напоминать», повтор, дату/время и заголовок — тоже кнопками.

**⏸ Пауза** открывает выбор срока (1 час / 1 день / 3 дня / 1 неделя /
2 недели / 1 месяц) или свой вариант текстом (например `2d`, `1w`, `12h`).

Напоминания нумеруются **внутри чата** (№1, №2, …), независимо от внутренних
id в базе.

**Разовое напоминание нельзя поставить в прошлое** — при вводе прошедшей даты
бот попросит ввести будущую. Для повторяющихся дата события может быть в
прошлом (например, дата рождения для ежегодного напоминания) — оно всё равно
сработает в ближайшую годовщину.

В `/add` бот сам создаёт получателя для чата. В групповом чате получателем
становится сама группа — напоминание получат все участники.

Отключить бота, оставив только REST API: `TELEGRAM_BOT_POLLING=false`.

## Смещения (offsets)

Строка `<число><единица>` или число секунд. Единицы: `s`, `m`, `h`, `d`, `w`.
Примеры: `"0"` (в момент события), `"30m"`, `"2h"`, `"1d"`, `"1w"`.
Каждое смещение — отдельное напоминание до события.

## Добавить новый канал доставки

1. Создайте класс-наследник `NotificationChannel` (см. `app/channels/email.py`
   как шаблон), реализуйте `send(address, message)`.
2. Добавьте нужные настройки в `app/config.py`.
3. Зарегистрируйте канал в `app/channels/__init__.py`:

```python
from .email import EmailChannel
register(EmailChannel())
```

Больше нигде менять ничего не нужно — получатели с `channel="email"` заработают.

## Настройки (.env)

| Переменная | Назначение | По умолчанию |
| --- | --- | --- |
| `TELEGRAM_BOT_TOKEN` | Токен бота (для канала telegram) | — |
| `DATABASE_URL` | Строка подключения к БД | `sqlite:///./data/reminders.db` |
| `POLL_INTERVAL_SECONDS` | Период опроса планировщика | `30` |
| `MISSED_GRACE_SECONDS` | Досыл пропущенного после простоя | `21600` (6 ч) |
| `DEFAULT_SNOOZE_DAYS` | Пауза по умолчанию для snooze | `30` |
| `LOG_LEVEL` | Уровень логирования | `INFO` |
